# ADR Checklist

## Stage 1 — Write
- [ ] File: `docs/adr/<sequence>-<title>.md`
- [ ] Status: Proposed
- [ ] Context explains why decision was needed
- [ ] Decision precise (not vague)
- [ ] Consequences covers trade-offs
- [ ] Acceptance test is concrete and runnable

## Stage 2 — Docs
- [ ] MASTER_IMPLEMENTATION_PLAN.md updated if plan changes
- [ ] Relevant runbook updated if operational behavior changes

## Stage 3 — Verify
- [ ] Acceptance test executed and passes
- [ ] Implementation fixed if test failed

## Stage 4 — Commit
- [ ] ADR Status updated to Accepted
- [ ] Commit: `docs(adr): <sequence>-<title> — verified and closed`
